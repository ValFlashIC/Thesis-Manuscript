\subsection{PHI}
\label{sec:soa:phi}

The authors of PHI~\cite{Mukkara2019_PHI} note that many large graph algorithms favour pull implementations (each node reads inputs from its neighbours) compared to push implementations (each node propagates results to its neighbours), since in a traditional memory system, push implementations require read-modify-writes which require accessing complete cache lines, even though a small unit of data is modified. With a traditional cache, the data access patterns make limited benefit of spatial and temporal locality. Although the authors focus on graph algorithms, they include SpMV in their evaluation, where the push approach corresponds to the \algo{OP}.

PHI is essentially a modified cache which is optimized to efficiently process commutative scatter updates. It stores, and coalesces update operations in the cache. When the cache overflows, PHI can switch to update buffering (UB). With UB, updates are deferred, by grouping them into bins, and streaming the requests to main memory. These streamed writes are efficient and when the updates are later processed, they can be performed efficiently because the binning process created spatial locality. Three new features are introduced, compared to a standard cache:

\begin{itemize}
    \item \emph{In-cache update storage and coalescing}. When an update for a line not in the cache is received, it does not fetch the line, but rather marks the line as being in update mode, and simply stores the update. If there are subsequent updates that hit the line, they are stored, or combined, using the commutative operator. When a cache line is evicted, if it contains multiple updates, PHI assumes that the coalescing has been effective, and it processes the operations via a read-modify-write from main memory (Figure~\ref{fig:phi_coalesce}).
    %
    \item \emph{Output Batching}. If a cache line is evicted, but contains few updates, PHI batches these updates as (address, value) tuples, allocating a new cache line to assemble the batch. Groups of updates are organized by \emph{bins} corresponding to a small memory region. The batches are streamed to main memory (Figure~\ref{fig:phi_ub}). Later, the deferred updates are read back and processed. The reading of the batches is efficient, as they are contiguous.
    %
    \item \emph{Reduced Cache Synchronization Traffic}. In multi-core systems, private caches act as local buffers, coalescing the updates and reducing coherency protocol traffic.
\end{itemize}

This combination of in-cache coalescing and update buffering enables PHI to take advantage of locality when possible (like Coup~\cite{ZHANG_2015__Exploiting_Commutativity}), but with the addition of UB, PHI improves memory utilization, even when the data being updated is too large to fit in the cache.

For SpMV, the authors report that much of the benefit comes from the reduction in memory synchronization traffic.
