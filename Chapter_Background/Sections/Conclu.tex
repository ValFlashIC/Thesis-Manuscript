\section{Conclusion}
\label{sec:bg:conclu}

\lettrine{I}{n} this chapter, we established the foundation for understanding the relationship between applications working with \gls{irrData} and the underlying challenges in hardware.
\Glspl{irrAccess} are common in sparse computing and cause reduced cache efficiency, limited parallelism, and higher memory latency exposure.
We analysed the most representative algorithms for sparse matrix multiplication kernels, and identified the operation that provoke \glspl{irrAccess}.
\Glspl{intersection} and \glspl{reduction} on \gls{irrData} with classical systems are costly, thus they represent the central challenge for efficient computation on sparse data.
To improve the performance of these operations, specialized hardware has been proposed as will be seen in the next chapter.

\bigskip\bigskip

\begin{highlight}{Takeaways}{Mulberry}
    \colorbf{Mulberry}{Sparse matrices:} large, highly sparse, structured (such as \gls{banded} matrices)

    \bigskip
    \colorbf{Mulberry}{Sparse formats:} diverse, reduce memory footprint
    \begin{itemize}
        \item \acrshort{coo}, the simplest
        \item \acrshort{csr} and \acrshort{csc}, the mostly used
        \item \acrshort{bacsc}, our own \gls{spFormat} for \gls{banded} matrices
    \end{itemize}

    \bigskip
    \colorbf{Mulberry}{Sparse kernels:} sparse \acrshort{mv} and \acrshort{mm} multiplications (such as \acrshort{spmv} and \acrshort{spmspm}), generate indirect accesses

    \bigskip
    \colorbf{Mulberry}{Algorithms:} \acrfull{ip}, \acrfull{op} and \acrfull{gust}, extended with \gls{tiling} methods
    \begin{itemize}
        \item Best potential to reduce memory transfers: \acrshort{gust}, followed by \acrshort{op} at low density
    \end{itemize}

    \bigskip
    \colorbf{Mulberry}{Problem:} \glspl{irrAccess}, generate high memory transaction latencies

    \bigskip
    \colorbf{Mulberry}{Challenges:}
    \begin{itemize}
        \item \gls{gather}: \gls{intersection} searches
        \item \gls{scatter}: ``random'' \glspl{reduction}, or storing and \emph{merging} \acrfullpl{po}
    \end{itemize}
\end{highlight}
