% Many recent high performance sparse accelerators (SpArch~\cite{Zhekai2020_SpArch}, Gamma~\cite{Zhang2021_Gamma}, ASA~\cite{Zhang2022_ASA}, Flexagon~\cite{Munoz-Martinez2023_Flexagon})
% support different algorithms to process the kernel, such as
% \emph{inner-product} (\algo{IP}), \emph{outer-product} (\algo{OP}) and \emph{Gustavson} (\algo{Gust})
%  algorithms~\cite{Faddeev1981_InnerOuter, Gustavson1978_GustAlgo}, obtaining better performance with \algo{OP} and \algo{Gust}.
% In theory, \algo{OP} and \algo{Gust} require fewer memory accesses than \algo{IP}. However, they require performing
% efficient \emph{reduction} operations on the output vector ($c_{key} \acc
% value$) which is accessed in a non-consecutive fashion that depends on the
% matrix structure.
% Indeed, when processing a SpMV ($A.b = c$) calculated as
% $\forall (m, n) \in \ldb 0, M \ldb \times \ldb 0, N \ldb, c_m \acc A_{m,n} \times b_n$,
% the \algo{OP} basically consists in iterating over $\ldb 0, N-1 \rdb$ first, while accumulating $c_m$ out-of-order.
% With a conventional memory system, these \emph{reductions} cause
% excessive memory traffic. With a specialized \emph{reduction cache}, this
% memory traffic is reduced, making \algo{OP} solutions very attractive.
